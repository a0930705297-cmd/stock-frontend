## 本輪重點

本輪主要集中在三個區塊：

1. 籌碼掃描與 `index.html` 前後端對齊
2. `即時資金雷達 PRO` 的 `/flow/*` 後端與前端連動
3. Discord Webhook 推播邏輯、時間規則與首頁文案整理

---

## 本輪已完成修改

### 1. `index.html` / `/chip_scan`
- 補回左側「籌碼掃描」區塊
- 新增 `chipCustom` 自訂股票代號輸入
- `runChipScan()` 改為配合新版 `/chip_scan?codes=...&min_foreign=...`
- 掃描來源改為：監控清單 + 常用股票 + 自訂輸入

### 2. `main.py` 的 `/chip_scan`
- 改成接收 `codes`、`min_foreign`
- 逐檔用 FinMind 掃描外資＋投信雙買超
- 回傳 `data`、`scan_date`、`total`、`note`

### 3. `籌碼分析.html`
- 修正「載入籌碼資料中...」容易卡住的問題
- `/chips/{code}` 作為主查詢
- `/ticker/{code}` 加短 timeout，失敗就 fallback，不再拖住整頁

### 4. 即時資金雷達後端
- 補成上市＋上櫃都可掃描
- `MIS` 代碼依市場自動組 `tse_` / `otc_`
- 修正成交量欄位優先用 `v`
- `flow_summary()` 快取補存 `_stock_data`
- `flow_industry()` 不再只剩 summary 前 5 檔資料
- `/flow/stock/{code}` 補上最近一筆法人欄位到 `latest`

### 5. `即時資金雷達Pro.html`
- 修正前後端欄位契約不一致
- 個股頁現在能顯示最近法人籌碼
- 產業頁移除不存在欄位的顯示
- 「較上次」改為使用後端 `prev_change`
- 文案改成以「產業」為主，不再混用不存在的 `theme`

### 6. Discord Webhook 功能
- `main.py` 已加入 `DISCORD_WEBHOOK` 環境變數支援
- 新增 `send_discord()`
- 新增 `/flow/monitor`
- 新增 `/flow/test_discord`
- 修正推播邏輯：
  - 不再用阻塞式 `time.sleep`
  - 只有成功送出才記錄 `_alerted`
  - `_alerted` 只保留目前與上一個 slot
  - `/flow/monitor`、`/flow/test_discord` 改用 `POST`

### 7. 時間規則與推播規則
- `flow` cache 改成 **3 分鐘**
- 盤中與盤後都用 3 分鐘資料節奏
- **盤後不推 Discord**
- 僅盤中 `09:00–13:30` 自動推播
- Discord 訊息已拆開顯示：
  - `資料時間`
  - `推播時間`

### 8. `即時資金雷達Pro.html` UX 強化
- 新增 Discord 狀態列
- 設定會存到 `localStorage`
- 刷新後會同步顯示 Discord 檢查結果
- 顯示狀態包括：
  - 已啟用
  - 已暫停
  - 檢查中
  - 已推播
  - 無新警示
  - 非盤中已跳過
  - Webhook 未設定

### 9. `home.html`
- 首頁卡片文案改成 `即時資金雷達 PRO`
- 描述更新為：
  - 上市＋上櫃
  - 產業排行
  - 個股 TOP 榜
  - Discord 推播
- 使用提示補上 `DISCORD_WEBHOOK` 需在 Railway Variables 設定
